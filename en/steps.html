The TutorialREST_Client application contains a simple message flow, which will call the REST API using a RESTRequest Node.

The SharedLibrary contains resources which may be referenced by different applications or other libraries.

The RESTRequest_API has one implemented operation (GET).

Please note that the RESTRequest_API contains its own copy of the swagger document, which desribes the REST API. The application references a copy of the swagger file in the SharedLibrary.

Prepare the IIB runtime

In order to run REST APIs in your Integration Server, you will need to enable The Cross-Origin Resource Sharing (CORS).

  1. In your Integration Console run the following command:
    mqsichangeproperties TESTNODE_userid -e default -o HTTPConnector -n corsEnabled -v true
    where
    TESTNODE_userid is the name of your Integration Node
    and
    default is the name of your Integration Server.
  2. In the Integration console window, stop the default integration node by typing the following command:
    mqsistop TESTNODE_userid
  3. Start the default integration node by typing the following command:
    mqsistart TESTNODE_userid

Follow these steps to complete the tutorial

The imported artifacts are all ready for you to test:

  1. Deploy RESTRequest_API by dragging and dropping it on to your integration server
  2. Inside the TutorialREST_Client application, open the REST_getUser message flow, and click the Flow Exerciser icon to start recording the message path through the flow.
  3. Click the Send Message icon to select a message to send to the flow.
  4. Choose the 'User 1' message, and click Send. Your request message is sent to the HTTPInput node.
    • The dialog will show you the input XML message structure: <User><UserNumber>1</UserNumber></User>
    • The dialog will show you the output JSON message structure: {"userNumber":"1","firstName":"Peter","lastName":"Pan","userDetails":"Employee"}

Close the dialog and the message path is automatically highlighted on the message flow. Click on any connection to see the data that passed through that connection.

Feel free to test the REST API by sending 'User 2' or 'User 3' test messages, following the above instructions.
