Import projects

When you ready to proceed with this tutorial use Import button to the right. This will import three artefacts in your Workspace - RESTRequest_API, SharedLibrary and an application TutorialREST_Client.

The RESTRequest_API has one implemented operation (GET). The TutorialREST_Client application contains a simple message flow, which will call the REST API using a RESTRequest Node. The SharedLibrary contains resources which may be referenced by different applications or other libraries.

Please note that the RESTRequest_API contains its own copy of the swagger document, which desribes the REST API. The application references a copy of the swagger file in the SharedLibrary.

Imported projects

In the Application Development view of your workspace the imported resources will be shown. This includes the RESTRequest_API, TutorialREST_Client application, SharedLibrary and a BARs folder, which contains the BAR file.

The BAR file has all resources packaged and ready for deployment. Alternatively, you can deploy the resources individually to your Integration Server. However, if the resources are deployed individually, it is required that the SharedLibrary is deployed before the RESTTutorial_Client application, as it references the Shared Library.

Once all artefacts have been deployed your application is ready to process messages. Use the 'Deploy' button on the 'Run' tab to complete the deployment.

Follow these steps to complete the tutorial

The TutorialREST_Client application is shown in the Application Development view of your workspace.

  1. In order to run REST APIs in your Integration Server, you will need to enable The Cross-Origin Resource Sharing (CORS).
    • In your Integration Console run the following command:
      mqsichangeproperties TESTNODE_userid -e default -o HTTPConnector -n corsEnabled -v true
      where
      TESTNODE_userid is the name of your Integration Node
      and
      default is the name of your Integration Server.
    • In the Integration console window, stop the default integration node by typing the following command:
      mqsistop TESTNODE_userid
    • Start the default integration node by typing the following command:
      mqsistart TESTNODE_userid
  2. Open the REST_getUser message flow, and click the Flow Exerciser icon to start recording the message path through the flow.
  3. Click the Send Message icon to select a message to send to the flow.
  4. Choose the 'User 1' message, and click Send. Your request message is sent to the HTTPInput node.
    • The input XML message is parsed from the HTTPInput node and the parameter 'UserNumber' with a value '1' is passed to the RESTRequest node.
    • The RESTRequest node calls the 'GET' operation of the RESTRequest_API
    • The implementation of the operation in this tutorial uses a Mapping node to map the input user number and return the user's first name, last name and its details.

After the request message is processed, the message path is automatically highlighted on the message flow.

Click on any connection to see the data that passed through that connection. You can see that the request (input) message data is in XML format and has one field named UserNumber. The response (output) message has been converted to a JSON format and has fields named 'firstName', 'lastName', 'userDetails', 'userNumber'.

Feel free to test the REST API by sending 'User 2' or 'User 3' test messages, following the above instructions.
